# CookDex 프로젝트 — 백업용 프롬프트

> **용도**: 지금까지 적용된 모든 작업을 한 문서에 모아, 프로젝트 복원·인수인계·다른 AI/팀원에게 맥락 전달 시 사용합니다.  
> **관련 문서**: 루트 `README.md`(다른 환경에서 작업하기), `docs/GEMINI_정리본_프롬프트.md` (Gemini 채팅에 붙여넣기용)

## 복사용 프롬프트 (전체 맥락)

아래 블록 전체를 복사해 새 채팅·다른 도구에 붙여넣으면 "지금까지 한 작업" 맥락을 그대로 이어받을 수 있습니다.

```
당신은 React Native(Expo) + Firebase 기반 요리/레시피 앱 "CookDex" 프로젝트의 후속 작업을 돕는 AI입니다. 아래는 지금까지 적용된 작업 전부와 디자인/기술 스택 정리본입니다. 이 맥락을 유지한 채로 요청사항을 구현해 주세요.

---

### 1. 프로젝트 개요
- **앱 이름**: CookDex (쿡덱스)
- **스택**: React Native, Expo (Expo Router), Firebase Auth + Firestore
- **주요 기능**: AI 레시피 생성, 스캔 레시피, 레시피 검색(Search), 저장 레시피(스크랩), 퀘스트, 요리 광장(플라자), 혜택 상점(포인트), 설정(프로필/맞춤 설정)
- **라우팅**: app/(tabs)/ 가 탭 레이아웃. 탭 바 노출: plaza, index=홈, benefits, profile=설정. recipes, quest, ranking, menu, create 등은 href: null로 탭 바에 숨김. 루트 Stack: login, (tabs), scanner, tutorial, benefits, search.

---

### 2. 디자인 시스템 (반드시 준수)
- **파일**: `constants/design-tokens.ts`
- **컬러**: primary: #E85D04 (메인 오렌지), primaryLight: #F97316, primarySoft: #FFF7ED, accent: #FB923C. bgMain: #FAFAF9, bgElevated/bgCard/bgModal: #FFFFFF. textMain: #1C1917, textSub: #78716C, textInverse: #FFFFFF. glassBg, glassBorder (글래스모피즘). actionShop, success, warning, danger, border 등.
- **스타일 톤**: 2026년형 모던, 밝은 배경 + 오렌지 포인트, 미니멀·글래스모피즘, 이모지 사용 최소화.
- **사용**: 새 화면/컴포넌트는 가능한 한 `Colors`, `Radius`, `Shadows`를 import해 일관된 룩 유지.

---

### 3. 설정(Profile) 탭
- **경로**: `app/(tabs)/profile.tsx`, 탭 제목 "설정".
- **설정 메인**: "설정" 타이틀 + 로그아웃. 개인 정보 행(아바타·이니셜·연필 배지·이메일) 탭 시 개인 정보 관리 모달. 퀵 카드 3개(레시피·스크랩·포인트). CookDex Plus 카드. 맞춤 설정(양념장·식단 목표·알레르기) — 서브뷰로 진입(별도 라우트 아님, settingsSubView). 요리 모드(음성·화면 꺼짐 방지). 기타(혜택 상점·튜토리얼·법적 고지).
- **서브뷰**: condiments | diet | allergies. "‹ 설정" 뒤로가기, "저장하고 설정으로 돌아가기". Android BackHandler 시 설정 메인으로만 복귀.
- **개인 정보 모달**: 프로필·표시 이름·기본정보·부가정보·칭호. "칭호 변경" 시 칭호 장착함 모달(titleModalVisible).
- **user 초기값**: useState(() => auth.currentUser) 로 로그인된 상태에서 설정 탭 진입 시 로그인 화면이 잠깐 안 보이도록.

---

### 4. 앱 파일 구조 (app/)
- **루트**: `app/_layout.tsx` — Stack(login, (tabs), scanner, tutorial, benefits, search). 비로그인 시 탭 진입 → /login. 로그인 후 탭 아닌 화면 → /(tabs) 리다이렉트(예외: scanner, create-recipe, tutorial, benefits, search). 튜토리얼: cookdex_has_agreed !== 'true' → /tutorial.
- **탭**: `app/(tabs)/_layout.tsx` — 노출: plaza, index, benefits, profile. 숨김: recipes, quest, ranking, menu, create 등.
- **스크린**: index(홈·레벨 뱃지·칭호·테마 모달·검색 버튼→/search·오늘 냉장고 파먹기→create-recipe), search(레시피 검색), plaza, recipes, quest, ranking, profile, benefits, menu, create.
- **탭 밖**: login, tutorial, scanner, create-recipe, benefits(모달), modal. Firebase: 루트 firebaseConfig.js. app 직하위 → ../firebaseConfig, (tabs) 직하위 → ../../firebaseConfig.

### 7. AI 레시피·테마·3가지 추천·광장·홈·검색 (적용된 작업 상세)

**7-1. 홈 테마 모달**
- 테마 카드 탭 → 테마 재료 입력 모달(openThemeModal). transparent, slide, 오버레이 탭 시 닫기. "직접 입력하여 레시피 만들기" 칩 THEME_MODAL_INGREDIENTS.slice(0,5). 저탄고지 카드 이미지 Unsplash URL.

**7-2. 테마 플로우 취소 시 홈**
- isFromThemeFlow = params.directStyle && params.directIngredients. 추천 모달 닫기/뒤로 시 router.replace('/(tabs)').

**7-3. 3가지 맞춤 요리 제안 (create-recipe) + 3D Cover Flow**
- showCurationPhase로 로딩·카드·제작 중·완성 레시피 동일 꽉 찬 화면. 오렌지 그라데이션+파도(reanimated). 카드 캐러셀: 스냅, CURATION_CARD_WIDTH 0.72*SCREEN_WIDTH. paddingHorizontal = (curationCarouselWidth - CURATION_CARD_WIDTH)/2, 슬롯 margin 없음. progress = index - curationScrollX/CURATION_SNAP, perspective 900, rotateY/translateX, Animated.View, 그림자·elevation 제거, BlurView·플래시 방지. THEME_COPY_TEMPLATES·pickThemeCopyForTitle, curationCardCopy. 레벨 상한 제거·하단 퍼센트 제거. __DEV__ 목업(curation_themes, recipe_markdown).

**7-4. 저장·광장 공유**
- 저장: cookdex_saved_recipes AsyncStorage. 광장: Firestore global_recipes, setDoc, 로그인 필요 시 알림, 성공 시 router.push('/(tabs)/plaza').

**7-5. AsyncStorage** — cookdex_legal_agreed, cookdex_theme_used_log, cookdex_draft_recipe 등.

**7-6. 홈 레벨 뱃지·칭호·인사**
- cookdex_user_exp, cookdex_equipped_title. calculateLevel: 500 EXP 후 200당 +1레벨. 원형 뱃지(levelBadgeRing #FED7AA, levelBadgeInner, 왕관 12시). titlePill(칭호). 인사 "{userName}님 어서오세요!".

**7-7. 홈 히어로**
- MaterialCommunityIcons fridge-outline 28. HERO_BG_IMAGE hero-bg-fresh-vegetables.png.

**7-8. 레시피 검색(Search) + 홈 진입**
- app/search.tsx: 내 주방(cookdex_saved_recipes)·광장(global_recipes+더미) 검색, 최근·인기 검색어(cookdex_search_history), 안심 필터, TTS 조리(expo-speech), 마크다운(react-native-markdown-display), 쿠팡 링크, 레시피 요청 시 global_recipes type "request" 후 plaza 이동.
- 홈 상단 "재료나 요리명 검색" 탭 → router.push('/search'). _layout.tsx에 Stack.Screen name="search", 리다이렉트 예외 segments[0] !== 'search'. Firebase ../firebaseConfig, AsyncStorage cookdex_ 접두사 유지.
```

## 사용 방법
- **다른 환경 셋업**: 루트 **README.md**의 "다른 환경에서 작업하기"를 따라 clone → npm install → .env 생성 → npx expo start 하면 됩니다.
- **백업**: 이 파일을 프로젝트 외부(클라우드·다른 저장소)에 복사해 두면, 나중에 "지금까지 뭘 했지?" 복원용으로 쓸 수 있습니다.
- **인수인계**: 위 "복사용 프롬프트" 블록 전체를 복사해 새 AI 채팅이나 팀원에게 주면, 맥락 없이도 이어서 작업할 수 있습니다.
- **Gemini 전용**: 짧게 쓰고 싶으면 `docs/GEMINI_정리본_프롬프트.md`를 사용하고, 상세 백업이 필요하면 이 파일을 사용하면 됩니다.
