### 1. 프로젝트 개요
- **앱 이름**: CookDex (쿡덱스)

**7-14-1. 기본 계량 가이드 버튼 위치 통일 (모든 레시피에 노출)**
- **의도**: "기본 계량 가이드"를 **모든 레시피**에서 동일한 위치(필요한 재료 바로 아래, 조리 순서 바로 위)에 노출.
- **app/(tabs)/recipes.tsx**: 상단에 있던 "기본 계량 가이드" 버튼 및 계량표 모달 제거.
- **app/recipe-detail.tsx**: 구조화된 레이아웃에서 "필요한 재료" 블록 다음·"조리 순서" 블록 전에 "기본 계량 가이드" 버튼 추가. 비구조화(마크다운만) 레시피는 TTS pill 아래·본문 위에 동일 버튼 노출. 탭 시 "쿡덱스 기본 계량표" 모달(1큰술/1작은술/1컵/1꼬집/약간/한 줌) 표시.
- **app/create-recipe.tsx**: AI 완성 레시피 블록에서 "필요한 재료" 섹션 다음·"조리 순서" 섹션 전에 동일한 "기본 계량 가이드" 버튼 추가, 같은 계량표 모달 연동.

**7-14-2. 레시피 상세(recipe-detail) 고도화 — 재료 구분·TTS 문구·신고·요리 후기**
- **재료 구분(만개의레시피 스타일)**: `## 필요한 재료` 파싱 시 **[ 재료 ]** / **[ 양념 & 조미료 ]** 로 구분. SEASONING_KEYWORDS 휴리스틱으로 양념/조미료 판별, 2열 시트로 각각 표시.
- **TTS 버튼 문구**: "🔊 조리 과정을 소리로 듣기" 로 통일.
- **요리 광장 레시피 상세**: 헤더 우측 **⋮** 메뉴 → "신고하기" → 모달(사유 선택 + 내용 필수) 제출 시 Firestore `reports` 컬렉션에 recipeId, reporterId, reason, detail, status 저장.
- **요리 후기 섹션**(source === 'plaza' 일 때만): 레시피 본문 아래에 표시.
  - Firestore `global_recipes/{recipeId}/reviews` 서브컬렉션 로드(createdAt 내림차순). 레시피 문서의 ratingAvg, reviewCount 사용(없으면 후기 배열로 계산).
  - 상단: 평균 별점(예: 4.2 ★★★★☆) + (N개). 포토 후기(photoUrl 있는 것) 가로 스크롤. 후기 리스트: 닉네임, 별점, 텍스트, 날짜, 후기별 **⋮** → 신고( review_reports 컬렉션).
  - "요리 후기 남기기" 버튼 → 모달: 별점 1~5, 후기 내용(필수). 부적절 언어 간단 필터(INAPPROPRIATE_WORDS) 제출 전 체크. 제출 시 runTransaction으로 reviews 서브컬렉션 추가 + global_recipes 문서의 reviewCount·ratingAvg 갱신.

**7-12. 혜택 상점(benefits) UI 리팩터링**
- app/(tabs)/benefits.tsx: 배경 Colors.bgMain, 상단 헤더·뒤로가기 버튼 라이트 톤. 포인트 대시보드(Colors.primarySoft + 🪙 아이콘)와 상품 카드(2열 그리드, Colors.bgElevated + Radius.xl + Shadows.glass)로 구성. 교환 버튼은 포인트 충분 시 Colors.primary, 부족 시 회색 비활성화.

**7-13. 스캐너(scanner) UI 리팩터링 (AR 가이드 + 크롭)**
- app/scanner.tsx: 카메라 오버레이를 라이트 테마로 개편. 중앙 AR 코너 가이드(centerGuide + corner*), 상단 우측 "오늘 스캔: N회 남음" 뱃지, 하단 글래스 바텀 시트(bottomMask) + 오렌지 원형 촬영 버튼(captureButton).  
  "재료 추가 / AI 레시피" 버튼은 동일 위치에 렌더링하되 opacity/pointerEvents로만 토글해 레이아웃 점프 제거.  
  촬영 후에는 manipulateAsync에서 중앙 가이드 영역만 crop 후 리사이즈해 썸네일이 가이드와 일치하도록 개선. 삭제 X 버튼 위치 조정(top: 0)으로 잘림 현상 제거. 안내 문구를 "사진을 터치해 식재료를 기입해주세요!"로 변경.

**7-15. 맞춤 요리 카드 탭 시 인분 선택(True 3D Flip + 인분 선택)**
- create-recipe.tsx: 카드 탭 시 카드 **뒷면**에 인분 선택(1~4인분 2×2 + "5인분 이상" + 취소). 인분 선택/취소 시 즉시 flippedCardIndex·pendingThemeForServings 초기화.
- **True 3D Flip**: 커버플로우용 3D transform은 바깥 카드 컨테이너에 유지하고, 그 안쪽에 앞/뒷면 레이어를 하나 더 두어 `flipProgress` 기반 `rotateY` 0→180 / -180→0 으로 **앞면/뒷면이 실제 180도 회전**. 앞면/뒷면 각각에 `perspective: 1000` + `backfaceVisibility: 'hidden'` + zIndex 스위칭을 적용해 안드로이드에서도 텍스트가 뒤집히지 않고 자연스럽게 보이도록 구성.

- **통합 레시피 상세 페이지(recipe-detail.tsx) 고도화**
  - 요리 광장/내 주방 공통 상세 페이지에 **구조화된 레이아웃** 적용:  
    - `## 필요한 재료`: `이름 | 수량` 포맷을 2열 그리드 + 가운데 세로 구분선 + 행별 하단 보더로 표시, 긴 텍스트는 아래 줄로 자연스럽게 배치.  
    - `## 조리 순서`: 번호 뱃지(오렌지 원형 + 흰색 숫자) + 오른쪽 본문 텍스트.  
    - `## 요리 팁(쿡덱스의 킥)`: 연한 오렌지 박스 내부에 `•` 불릿으로 표시.  
  - 하단 액션을 **아이콘 버튼 3~4개 그리드**로 개편:  
    - 요리 광장 진입 시: `저장하기`(내 주방 스크랩), `릴레이 제작`(해당 레시피+재료 기반으로 create-recipe 릴레이 모드 이동), `장보기`(재료 목록을 쿠팡 검색으로 연결하는 하단 모달).  
    - 내 주방 진입 시: `광장 공유`(FireStore `global_recipes`에 게시, 중복 공유 방지), `다시 제작`, `장보기`, `삭제하기`(AsyncStorage에서 제거 후 뒤로 이동).

- **AI 레시피 완성 화면(create-recipe.tsx) 버튼/장보기 + 중복 저장/공유 제어**
  - AI가 생성한 최종 레시피 표시 영역 하단에 아이콘 버튼 4개 배치:  
    - `저장하기`(내 주방), `광장 공유`, `다시 제작`, `장보기`.  
  - `장보기`: `## 필요한 재료` 섹션을 파싱해 재료명 리스트를 만들고, 모달에서 각 항목을 탭하면 쿠팡 검색 페이지(`https://m.coupang.com/nm/search?q=...`)로 이동.
  - **중복 방지 로직**:  
    - 저장: `cookdex_saved_recipes`의 `content` 기준으로 동일 레시피가 이미 있으면 `"이미 저장된 레시피입니다!"` 한 번만 알리고, 이후 저장 버튼은 무시.  
    - 공유: Firestore `global_recipes` 기준으로 같은 레시피를 여러 번 공유하려 하면 `"이미 공유 완료된 레시피예요!"` 한 번만 알리고, 이후 공유 버튼은 무시.  
    - 사용자가 내 주방에서 해당 레시피를 삭제한 뒤에는, 다시 저장하면 **새 레시피로 취급**(AsyncStorage 현 상태 기준으로만 중복 여부 판단).

- **내 주방(recipes.tsx) — 멀티 선택 삭제 UX**
  - 기존 단일 삭제 버튼을 **상단 우측 "삭제하기" + 위쪽 "전체/취소" 버튼 조합**으로 재설계:
    - 평소: 카드 탭 시 상세 진입, 우측에는 `삭제하기` 버튼만 노출.

      - `삭제하기` 버튼 **위쪽 줄**에 `전체`/`취소` 버튼이 나타나며, 항상 고정된 영역 안에서만 보이기 때문에 레이아웃 점프가 없음.  
      - `전체`: 현재 화면의 모든 저장 레시피를 선택/해제.  
      - `취소`: 삭제 모드 종료 및 선택 목록 초기화.  
    - 삭제 모드에서 `삭제하기` 버튼은 선택된 레시피들을 대상으로 `"선택한 N개의 레시피를 내 주방에서 삭제할까요?"` Alert를 띄운 뒤, 승인 시 AsyncStorage와 상태에서 모두 제거.

## 9. 최신 작업 요약 (릴레이·스타일 입력·셰프의 킥·Scanner DEV 목업)

- **요리 광장 릴레이 챌린지 연동**
  - Firestore `global_recipes`에 릴레이 메타데이터 필드 추가:  
    - `relayFromId`: 원본 레시피 ID,  
    - `relayRootId`: 릴레이 체인의 루트 레시피 ID,  
    - `relayDepth`: 루트에서 몇 단계 떨어진 릴레이인지(0, 1, 2…).  
  - `recipe-detail.tsx`에서:  
    - Plaza 레시피 로드 시 위 필드를 함께 세팅하고, 하단에 두 개의 섹션을 노출:  
      - **"해당 요리를 만든 다른 셰프들!"**: 비슷한 재료 조합과 완성 사진이 있는 다른 광장 레시피를 2열 카드 그리드로 표시(셰프 닉네임·요리 이름·별점·후기 수).  
      - **"{제목}에서 변형된 다른 셰프의 릴레이 요리!"**: `relayFromId === 현재 recipeId` 인 릴레이 레시피들을 2열 카드로 표시(닉네임·요리 이름·별점·후기 수), 카드 탭 시 해당 릴레이 상세로 이동.  
    - Plaza 상세 상단 액션에 **"릴레이 제작"** 버튼을 두고, 릴레이 제작 시 `relayParentId / relayRootId / relayDepth`를 create-recipe로 전달.
  - `create-recipe.tsx`에서 광장 공유 시:  
    - 광장용 ID(`plazaRecipeId`)를 매번 새로 발급해, 같은 레시피를 여러 번 공유해도 Firestore 문서가 덮어쓰이지 않도록 분리.  
    - 전달된 릴레이 파라미터가 있을 경우 `relayFromId / relayRootId / relayDepth`를 함께 저장.

- **AI 레시피 결과 화면(UI/메타 정보 정리)**
  - `create-recipe.tsx`의 AI 결과 모달 상단에:  
    - 마크다운에서 `# 제목`을 추출해 **굵은 요리 제목**(`generatedTitle`)을 노출.  
    - 제목 아래에 **인분/시간/난이도 pill 메타 태그**를 상세 페이지와 동일한 스타일(`metaRow`, `metaChip`)로 표시(현재 기본값 2인분/20분 이내/보통).  
    - 그 아래에 상세 페이지와 동일한 디자인의 **TTS pill 버튼** `"🔊 조리 모드로 듣기"` 배치.

- **스타일 입력 UX 개편 (텍스트 한 줄 → 구조화 폼)**
  - 기존: 한 줄 입력(`preferredStyle`)에 희망 스타일을 모두 적게 한 뒤, 문자열 파싱으로 dish/method를 추출.  
  - 변경: **"나만의 요리 스타일 추가!"** 모달을 도입해 3칸으로 분리 입력:  
    - `원하는 요리` (예: 라면, 비빔밥, 고등어 조림),  
    - `원하는 조리` (예: 볶음, 찌개, 튀김, 자작하게 끓이는 등),  
    - `스타일 추가` (예: 싱겁게 먹을 수 있는 미역국, 아이도 먹을 수 있게 매운맛은 거의 없이 등).  
  - 내부 로직:  
    - 3개 입력값을 `원하는 요리 이름: …`, `원하는 조리 방식: …`, `추가 설명: …` 형태로 합쳐 `preferredStyle`에 저장.  
    - `generateFinalRecipe`에서는 세 칸 값이 있으면 그대로 프롬프트에 구조화 블록(`--- 🧾 사용자 지정 요리 방향 ---`)으로 주입.  
  - 간단한 안전 필터:  
    - `checkStyleSafety`로 욕설/정치/종교/위험 조리 키워드(생닭 그대로, 세제/표백제 등)를 막고,  
    - 나중에 Gemini Pro에 붙일 수 있도록 `moderateStyleWithLLM` 래퍼를 추가(현재 dev에서는 mock으로 항상 'ok' 반환).
  - 동일한 스타일 모달을 **스캐너(scanner.tsx)의 "✨ AI 레시피" 플로우에도 공용**으로 적용해, 스캔 기반 레시피에서도 같은 UX로 희망 스타일을 받을 수 있도록 통일.

- **광장 공유 전 "셰프의 킥!" 팝업 & 완성 사진 재촬영**
  - `create-recipe.tsx`에서 광장 공유 버튼을 누르면:  
    1. 먼저 카메라로 **완성 요리 사진을 촬영**하고(로컬 URI만 저장),  
    2. 이후 **"셰프의 킥!" 모달**을 띄워 사진 미리보기 + 추가 정보를 입력받은 뒤,  
    3. 최종 확인(Alert `"이대로 공유하시겠습니까?"`) 후에야 Firestore `global_recipes`에 업로드/저장.  
  - 셰프의 킥 모달 구성:  
    - 상단 제목: `셰프의 킥!`  
    - 상단에 촬영한 사진 미리보기(3:2 비율, 90% 폭, 가운데 정렬).  
    - 사진 아래 중앙에 **"다시 찍기"** pill 버튼 → `"요리의 사진을 다시 찍겠습니까?"` 확인창 후 재촬영.  
    - 텍스트 입력 칸(멀티라인): placeholder `"추가/변경하신 사항을 자유롭게 적어주세요! 안 적고 제출하셔도 OK!"`.  
    - 하단:  
      - `이대로 광장에 공유하기` → 확인 Alert 후 Firestore에 `chefKick` 필드와 함께 저장,  
      - `취소` → `"이전 사항이 저장되지 않습니다. 정말 취소할까요?"` Alert 후 닫기.  
    - 모달 바깥(오버레이)을 탭해도 동일한 취소 확인 Alert가 뜨도록 처리.
  - 사진 업로드 최적화:  
    - 촬영 시에는 로컬 URI만 저장하고 모달을 즉시 띄워 **UX를 빠르게** 하고,  
    - Firestore 저장 직전에만 Storage 업로드를 수행(업로드 실패 시 로컬 URI라도 `recipePhotoUrl`에 저장하여 최소한 기기에서는 사진이 보이도록 함).

- **Scanner용 Gemini DEV 목업 추가**
  - `app/scanner.tsx`의 `callGeminiAPI`에 `__DEV__` 분기를 추가해, **개발 환경에서는 실제 Gemini 호출 대신 하드코딩된 JSON을 반환**:  
    - 텍스트 재료 → `invalid_items` + `detected_ingredients` + `curation_themes`(불고기덮밥/김치찌개/두부스테이크 등).  
    - 스캔 이미지 → `status: "SUCCESS"`, 예시 재료 리스트 + 3가지 테마.  
    - 비식재료 검사용 프롬프트 → `{ is_food: true }` 등의 간단 응답.  
    - 최종 레시피 생성 → 불고기덮밥 예시 마크다운/쇼핑 리스트.  
  - 이를 통해 **DEV 환경에서 Gemini API 키/쿼터와 상관없이 스캐너 → AI 레시피 플로우를 반복 테스트**할 수 있음. 운영 빌드에서는 `__DEV__ === false` 이므로 자동으로 실제 Gemini API를 사용.

## 10. 최신 작업 요약 (맞춤 요리 인분 선택·카드 플립·터치·릴레이 상세)

- **맞춤 요리 제안 카드 탭 시 카드 뒤집기 + 인분 선택**
  - **파일**: `app/create-recipe.tsx`
  - 카드 탭 시 **카드 뒤집기(플립)** 후 **인분 선택 UI** 표시: 1~4인분(2×2 그리드) + "5인분 이상" 버튼 + 취소.
  - 인분 선택/취소 시 **즉시** `setFlippedCardIndex(null)`, `setPendingThemeForServings(null)` 호출해, 인분 화면에서 돌아온 뒤 다시 탭해도 정상 동작.
  - 인분 선택에 따라 `generateFinalRecipe(theme, servings)` 호출 시 **재료/조미료·N인분·N분 이내·난이도**가 자동 반영되며, 결과 화면·광장 공유 메타(`finalServings`, `finalMinutes`, `finalDifficulty`)에 연동.

- **2D 플립 → opacity 크로스페이드**
  - 부모 카드에 이미 `transform`(perspective, rotateY, scale)이 있어 자식의 `scaleX` transform이 미동작할 수 있어, **transform(scaleX) 제거** 후 **opacity만** 사용하는 크로스페이드로 변경.
  - 앞면: `opacity: 1 - p`, 뒷면: `opacity: p` (Reanimated `cardFlipProgress` 0→1 withTiming). `FLIP_OPEN_DURATION`(380ms) 등 적용.

- **중앙 카드만 두 면 렌더 + 터치 안정화**
  - **중앙 카드만** 두 면(앞/뒤) 마운트: `currentCurationIndex === index` 일 때만 앞면·뒷면 `Animated.View` 렌더. 탭은 중앙 카드일 때만 플립 트리거.
  - **pointerEvents**: 뒤집힌 상태(`flippedCardIndex === index`)일 때 앞면 `pointerEvents='none'`(터치 통과), 뒷면 `pointerEvents='box-none'`(자식 버튼만 터치). 뒤집기 전에는 뒷면 `'none'`.
  - **카드 탭 인식 개선**: 카드용 `TouchableOpacity`에 `hitSlop={{ top: 12, bottom: 12, left: 12, right: 12 }}`, `delayPressIn={0}` 적용해 가장자리 터치도 탭으로 인식되도록 함.
  - 슬라이드로 다른 카드로 이동 시 `onMomentumScrollEnd`에서 `flippedCardIndex`/`pendingThemeForServings` 초기화, `cardFlipProgress.value = 0`.

- **레시피 상세(recipe-detail) — 릴레이 필드 로드 및 UI**
  - **파일**: `app/recipe-detail.tsx`
  - Plaza 레시피 로드 시 Firestore 문서의 **relay 필드**(`relayFromId`, `relayRootId`, `relayDepth`)를 함께 매핑해 상태에 보관.
  - **"해당 요리를 만든 다른 셰프들!"** / **"{제목}에서 변형된 다른 셰프의 릴레이 요리!"** 섹션: `relayChildren` 등 로드 후 2열 카드 그리드로 표시.
  - **릴레이 제작** 버튼 탭 시 `handleStyleChange` 등에서 `relayParentId`, `relayRootId`, `relayDepth`를 params로 담아 `create-recipe`로 이동.

- **create-recipe 광장 공유 시 릴레이 저장**
  - 릴레이 모드로 진입한 경우 params에서 `relayFromId`/`relayRootId`/`relayDepth`를 읽어, 광장 공유 시 Firestore `global_recipes` 문서에 해당 필드를 함께 저장.
